# UDEMY Subtitle Extension — Development Instructions

## Project Overview
Chrome MV3 extension that injects a learning panel into Udemy course pages for real-time AI-powered subtitle assistance and study tools. The extension integrates with a local LLM API and observes Udemy's video player DOM.

---

## Core Constraints (Non-Negotiable)

### 1. **Zero Manual Intervention**
- **Rule**: Never ask the user to manually open Chrome, navigate URLs, close windows, or perform setup steps.
- **Why**: The user explicitly stated "que lo hagas tu no me vuelvas a pedir participacion" (do it yourself, no manual requests).
- **Applied to**: Testing, deployment, debugging, configuration.
- **Exception**: Only ask if the task is structurally impossible to automate without user credentials or hardware interaction.

### 2. **Single AI Call Per User Action**
- **Rule**: Each user action in the extension triggers **at most 1 call** to the local LLM API.
- **Why**: Cost control and performance predictability.
- **Enforcement**:
  - `src/background.js` must NOT retry failed LLM requests automatically.
  - No cascade or chaining of API calls within a single user trigger.
  - Cache or batch LLM results when possible to avoid duplicate calls.
- **Example**: User clicks "Generate Learning Panel" → 1 API call to local LLM → return result or error.

### 3. **DOM Injection Pattern**
- **Rule**: The learning panel must be injected with `id="usg-learning-panel"` immediately below `.video-player--container--`.
- **Implementation**: See `src/content-script.js` → `ensureLearningPanel()` function.
- **Selector Guarantee**: `.video-player--container--` is Udemy's player wrapper; this is the stable injection point.
- **Never change this selector** without verifying Udemy's DOM structure across multiple courses.

### 4. **No Mock Testing for Core Flows**
- **Rule**: Integration tests must use **real Chrome instances** (via Playwright with `channel: 'chrome'`), not Chromium mock browsers.
- **Why**: Udemy's Cloudflare anti-bot detection rejects headless/automation-flagged browsers; real Chrome evades this.
- **Applied to**: End-to-end testing of DOM injection and API calls.
- **Exception**: Unit tests for utility functions may use jsdom or Node mocks if isolated from DOM.

### 5. **Error Resilience**
- **Rule**: Content script must **never crash**; wrap async operations in `try/catch`.
- **Pattern**:
  ```javascript
  async function init() {
    // ... setup code ...
    init().catch(() => {
      // Silently fail. Popup can show errors to user.
    });
  }
  ```
- **Rationale**: Extension must remain loaded even if LLM API is down or DOM observation fails.

## Architecture Rules

### File Structure
```
UDEMY-SUBTITLE/
├── src/
│   ├── content-script.js       # DOM observer, panel injection, message handler
│   ├── background.js            # Service worker, LLM API calls (max 1 per message)
│   └── popup.html / popup.js    # UI for settings and panel interactions
├── manifest.json                # MV3 manifest (load-extension, permissions)
├── token.txt                    # Session cookies (for testing only; not in version control)
└── .instructions.md             # This file
```

### Content Script (`src/content-script.js`)
- **Responsibility**: Observe Udemy DOM, detect video player, inject/manage learning panel.
- **Do NOT make LLM calls** directly; message `background.js` instead.
- **Storage**: Use `chrome.storage.local` for lecture-specific data (subtitles, panel state).
- **Observers**: MutationObserver to watch for `.video-player--container--` mount/unmount.

### Storage & Persistence
- **Storage Engine**: `chrome.storage.local` for all lecture-specific data.
- **Data Lifetime**: Keep panel content and subtitles for the last **30 lectures**. Archive older data (optional: export to user's Downloads before delete).
- **Cleanup**: On extension startup, scan storage and remove entries older than 30 days or exceeding 50 lectures cached.
- **Keys Convention**: Prefix all storage keys with `usg_` to avoid collisions (e.g., `usg_learning_panel_by_lecture_v1`, `usg_subtitles_by_lecture_v1`).

### Background Script (`src/background.js`)
- **Responsibility**: Handle LLM API calls, state management, message routing.
- **Constraint**: **Max 1 LLM call per message** from content script.
- **API Design**: 
  - Input: `{ type: "USG_GENERATE_LEARNING_PANEL", lectureId, subtitles }`
  - Output: `{ ok: true, panelContent, error: null }` or `{ ok: false, error: "reason" }`
- **No automatic retries**: If LLM call fails, return error immediately (fail-fast).
- **User-initiated retries**: Popup/panel MAY offer a manual "Retry" button for user to click; background.js will accept the retry message but still make only 1 API call per request.

### Manifest (`manifest.json`)
- **Version**: Manifest V3 only. Never V2.
- **Permissions**: Minimum needed (`scripting`, `storage`, `tabs`, `activeTab`).
- **Host Permissions**: Only `https://*.udemy.com/*` or specific lecture URLs.
- **Load Extension**: `--load-extension=<path>` when launching test browsers.

## Testing Strategy

### Automated E2E Testing
- **Tool**: Playwright with stealth plugin (`puppeteer-extra-plugin-stealth`).
- **Launch**: `chromium.launchPersistentContext(userDataDir, { channel: 'chrome', ... })`.
- **Session**: Inject valid cookies from `token.txt` into browser context.
  - **Token.txt Policy**: Temporary session tokens for testing only. NOT versioned. User provides fresh tokens before each test run.
  - **Token Expiry**: If tokens expire, test fails gracefully. User updates `token.txt` with new cookies from their logged-in Chrome session.
  - **Security**: Never commit `token.txt` to git. Add to `.gitignore`. Delete after testing completes.
- **Assertions**:
  1. Verify `.video-player--container--` renders.
  2. Verify `#usg-learning-panel` is injected below player.
  3. Verify LLM call happened and panel content updated.
- **Timeout**: Give tests **120+ seconds** for Cloudflare bypass and page load.

### Manual Testing (Last Resort)
- **Load Extension**: `chrome://extensions` → Developer Mode → Load Unpacked → select project folder.
- **Verify Locally**: Navigate to Udemy course in your personal Chrome, check panel appears.
- **Report Results**: Screenshot of panel injected into real course page.

## Common Workflows

### Adding a New Feature to Panel
1. **Design**: Sketch panel UI (colors, layout, interactions).
2. **DOM**: Add HTML/CSS to `generateLearningPanel()` in content-script.js.
3. **Message**: Define new message type (e.g., `USG_FETCH_SUMMARY`) in background.js.
4. **API**: Implement single LLM call to generate content (no retries, fail fast).
5. **Test**: Run E2E with real Chrome; verify 1 API call only.
6. **Deploy**: Update manifest version, commit.

### Debugging LLM API Failures
- **Check**: Is local LLM API running? (e.g., `http://localhost:8000/api/...`)
- **Check**: Do cookies in `token.txt` work? (test in personal Chrome)
- **Check**: Is Udemy blocking the request? (Cloudflare, anti-bot, session expired)
- **Action**: Log error in background.js console; let content script handle retry UI.

### Fixing DOM Injection Issues
- **Symptom**: Panel not appearing below video player.
- **Debug**: Inspect Udemy page → check if `.video-player--container--` exists.
- **Check**: MutationObserver firing? (add console.log in observer callback).
- **Fix**: May need to adjust selector if Udemy updated DOM. Use `semantic_search` in codebase or check Udemy's HTML structure via `fetch` MCP.

### Before Release

2. **Scan**: `snyk test` for CVEs in dependencies.
3. **Manual Test**: Load unpacked extension in Chrome, verify panel works on 2+ courses.
4. **Update Version**: Bump `manifest_version` number.

### Publishing
- **Chrome Web Store**: Package as `.zip`, upload with manifest and screenshots.
- **Distribution**: Share unpacked folder path for local testing; users load via `chrome://extensions`.

## Common Pitfalls & Solutions

| Pitfall | Solution |
|---------|----------|
| Panel injected but invisible | Check z-index, CSS visibility, parent overflow. Use DevTools to inspect computed styles. |
| LLM API called twice | Add request deduplication by `lectureId`. Check background.js message handler for cascades. |
| Cloudflare blocks test automation | Use `puppeteer-extra-plugin-stealth`, real Chrome channel, avoid headless flag. |
| Content script crashes on new Udemy layout | Wrap DOM selectors in try/catch; log errors instead of throwing. |
| Cookie injection fails in new profile | Ensure cookies include `Secure` flag, correct `domain`, and HTTPS URL. Use `{ url: 'https://www.udemy.com', ... }`. |

## Code Style & Conventions

- **Language**: JavaScript (ES6+).
- **Naming**: `camelCase` for functions/variables, `UPPER_SNAKE_CASE` for constants.
- **Comments**: Comment *why*, not *what* (code is self-documenting).
- **Error Handling**: Always provide context (e.g., `"LLM API failed: ${error.message}"`).
- **Storage Keys**: Prefix with `usg_` (e.g., `usg_learning_panel_by_lecture_v1`) to avoid collisions.

# Udemy Subtitle Bridge — Agent Instructions

Use this repo as a Chrome MV3 extension for Udemy subtitle tooling. Keep changes small, local, and aligned with the current code rather than the older drafts.

## Start Here

- Read [README.md](README.md) for the current product scope and manual flows.
- Read [INSTALAR.md](INSTALAR.md) for the user-facing install and verification steps.
- For this project, the local API docs are critical: `http://127.0.0.1:8010/docs`, `http://127.0.0.1:8010/redoc`, and `http://127.0.0.1:8010/openapi.json`.
- Main runtime files are [src/content-script.js](src/content-script.js), [src/background.js](src/background.js), [src/popup.js](src/popup.js), and [src/page-network-bridge.js](src/page-network-bridge.js).
- Treat [src/content-script-old.js](src/content-script-old.js) and [src/content-script-v2.js](src/content-script-v2.js) as legacy references unless the user asks to restore behavior from them.

## Working Rules

- Do not ask the user to manually drive Chrome or the test flow unless automation is impossible.
- Keep extension actions to one local API call per user-triggered action. Do not add automatic retries in the background script.
- Preserve the non-blocking content-script pattern: panel injection and message wiring must stay resilient if Udemy changes or the API is unavailable.
- Keep storage keys prefixed with `usg_` and use `chrome.storage.local` for lecture-scoped state.
- Update the UI or DOM only through the existing Udemy injection path; the panel is expected below the video player container.

## Code Paths That Matter

- [src/content-script.js](src/content-script.js) owns panel injection, network capture wiring, and content-script state.
- [src/background.js](src/background.js) owns LLM/API orchestration and message routing.
- [src/popup.js](src/popup.js) owns user-facing controls and status refresh.
- [manifest.json](manifest.json) is MV3-only and already scopes access to Udemy plus the local API endpoints.

## Testing Expectations

- Prefer real Chrome for integration checks. The repo already contains Playwright-based scripts such as [test-real-udemy.js](test-real-udemy.js), [test-full-workflow.js](test-full-workflow.js), [run-final-test.js](run-final-test.js), and [stealth-test.js](stealth-test.js).
- Use the existing `token.txt` flow only for local testing; do not treat it as a durable fixture.
- For quick syntax checks, validate the touched JS files with `node --check` before broader testing.

## Common Failure Modes

- If the panel does not appear, inspect the Udemy player DOM and the injection path in [src/content-script.js](src/content-script.js) before changing unrelated code.
- If a translation or panel-generation flow doubles calls, check [src/background.js](src/background.js) for cascaded message handling or retry logic.
- If network capture misses subtitles, check [src/page-network-bridge.js](src/page-network-bridge.js) and the listener setup in [src/content-script.js](src/content-script.js).
